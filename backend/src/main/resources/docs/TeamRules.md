# 🧑‍💻 Teamregeln – AI-Deskhelp

Diese Regeln sollen sicherstellen, dass wir strukturiert und ohne Chaos zusammenarbeiten.

---

## 🌿 Branching

- Der `main` Branch ist **immer stabil**
- **Keine direkten Commits auf `main`**
- Für jedes Feature, Bugfix oder Dokumentation wird ein eigener Branch erstellt

### Naming Convention:
- `feature/...` → neue Features  
- `fix/...` → Bugfixes  
- `docs/...` → Dokumentation  

### Beispiele:
- `feature/chat-ui`
- `feature/ticket-escalation`
- `fix/api-error-handling`
- `docs/readme-update`

---

## 🔀 Pull Requests (PR)

- Änderungen werden **immer über Pull Requests** gemerged
- Mindestens **eine andere Person schaut drüber**
- PR sollte enthalten:
  - kurze Beschreibung
  - was wurde geändert?
  - ggf. Screenshots (Frontend)

---

## 🧾 Commits

- Klare und verständliche Commit Messages
- Lieber mehrere kleine Commits als ein großer

### Beispiele:
- `feat: Chat UI hinzugefügt`
- `fix: Fehler bei API-Request behoben`
- `docs: README aktualisiert`

---

## 🤝 Zusammenarbeit

- Backend und Frontend müssen sich auf **API-Struktur** abstimmen
- Keine Breaking Changes ohne Absprache
- Wenn größere Änderungen geplant sind → vorher kurz abstimmen

---

## ✅ Definition of Done

Ein Task ist fertig, wenn:

- Code implementiert ist
- lokal getestet wurde
- keine offensichtlichen Fehler vorhanden sind
- Pull Request erstellt wurde
- Review erfolgt ist
